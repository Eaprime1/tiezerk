## Coding and Documentation Conventions

### File Naming
- Use lowercase, hyphen-separated: `quantum-forest-overview.md`
- Concept files: `[concept-name].md`
- Protocol files: `[PROTOCOL_NAME].md` (CAPS for protocols)

### Document Structure
Every environment/concept document should include:

```markdown
# [Concept Name]
## Identity
[One paragraph: what this IS in the universal sense]

## Properties
[Enumerable characteristics — what makes this unique]

## Relationships
[How this connects to other tiezerk environments, ashkharh entities, nexusiam]

## Navigation
[How to enter/traverse/exit this environment]

## Seeds Forward
[Open threads, emergent directions, what grows from here]
```

### Commit Message Format
```
[type]: [environment] — [brief description]

feat: quantum-forest — add field resonance properties
doc: spacetime-sea — navigation protocol draft
fix: void-marrow — wormhole link correction
```

### Branch Naming
```
feature/[environment]-[aspect]
doc/[environment]-[document]
fix/[environment]-[issue]
```
